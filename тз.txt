# Паспорт проекта: «ФинГоризонт»

### 1. Название и Брендинг
*   **Название проекта:** **ФинГоризонт** (ассоциация с планированием будущего, дальновидностью).
*   **Доменное имя:** `finhorizon.ru` (или `budget-scope.ru`, `fin-cast.ru`).
*   **Слоган:** «Ваш бизнес под четким контролем» / «Планируйте уверенно».
*   **Логотип:** Стилизованное изображение горизонта или восходящего графика, вписанного в круг (символ целостности и обзора).

### 2. Визуальный стиль (UI/UX)
Стиль должен быть строгим, «бухгалтерским», но современным.
*   **Цветовая тема:**
    *   **Основной:** Темно-синий (`#2C3E50`) — надежность, бизнес.
    *   **Акцентный:** Изумрудный (`#27AE60`) — прибыль, рост, положительные значения.
    *   **Фон:** Светло-серый (`#ECF0F1`) — для снижения нагрузки на глаза при работе с таблицами.
    *   **Ошибка/Убыток:** Приглушенный красный (`#E74C3C`).
*   **Шрифты:**
    *   Заголовки: **Roboto Condensed** (компактность для таблиц).
    *   Цифры: **Monospace** (например, `Courier New` или `Roboto Mono`) — для выравнивания чисел в колонках.
*   **Стиль интерфейса:** Dashboard (Панель приборов). Карточная система, четкие границы таблиц, минимализм.

### 3. Технический стек (исправленный)
Убираем Node.js, оставляем классический надежный стек для курсовой:
*   **Frontend:** HTML5, CSS3, JavaScript (Vanilla JS или jQuery для AJAX-запросов без перезагрузки страниц). Библиотека **Chart.js** для графиков.
*   **Backend:** **PHP 8.x** (обработка логики, CRUD операции, расчеты).
*   **База данных:** **MySQL** (хранение пользователей, сценариев, статей бюджета).
*   **Сервер:** Apache/Nginx (локально OpenServer или XAMPP).

### 5. Наполнение для Пояснительной Записки (Ключевые тезисы)

#### Введение
*   **Актуальность:** Малый бизнес часто ведет финансы в Excel, что неудобно для совместной работы и прогнозирования. Нужна легкая веб-система.
*   **Цель:** Разработка веб-приложения «ФинГоризонт» для составления бюджетов и прогнозов выручки.
*   **Задачи:** Спроектировать БД, реализовать CRUD на PHP, сделать визуализацию графиков, протестировать безопасность.

#### Глава 1 (Теория)
*   Опишите, почему **PHP** выбран для бэкенда (широкая поддержка, легкость деплоя, достаточная производительность для малых задач).
*   Сравните **MySQL** с другими СУБД.
*   В разделе 1.3 обязательно укажите требование **точности вычислений** (использование типов `DECIMAL` в БД вместо `FLOAT`, чтобы не терять копейки).

#### Глава 2 (Практика)
*   **2.1 Архитектура:** Опишите паттерн MVC (Model-View-Controller), даже если пишете на чистом PHP, это покажет глубину знаний.
*   **2.2 CRUD:** Опишите, как реализовано создание/редактирование статей бюджета через формы HTML и обработчики PHP.
*   **2.3 Прогнозирование:** Опишите логику: пользователь вводит плановые значения, система считает итог и рисует график (через JS библиотеку Chart.js, данные берет из PHP).
*   **2.4 Безопасность:** Защита от SQL-инъекций ( Prepared Statements PDO), хеширование паролей (`password_hash`).

#### Приложения (как в образце)
*   **Приложение 1:** Техническое задание (Функционал: создание бюджета, редактирование статей, удаление сценария, экспорт).
*   **Приложение 2:** Схема базы данных (Таблицы: `users`, `budgets`, `budget_items`, `categories`).
*   **Приложение 3:** Листинг кода (Контроллер бюджета, Модель подключения к БД).
*   **Приложение 4:** Скриншоты (Главная, Форма создания бюджета, График прогноза).

### 6. Структура Базы Данных (для Приложения 3)
Вместо таблиц форума и игр (как в образце), у вас будут:

1.  `users` (id, email, password, created_at)
2.  `budgets` (id, user_id, title, period_start, period_end, scenario_type)
3.  `categories` (id, user_id, name, type [income/expense])
4.  `budget_items` (id, budget_id, category_id, amount, comment, date)
