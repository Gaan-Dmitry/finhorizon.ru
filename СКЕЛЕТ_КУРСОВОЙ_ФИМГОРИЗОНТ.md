## ВВЕДЕНИЕ

### Актуальность темы исследования

В условиях активной цифровизации экономики малый и средний бизнес сталкивается с необходимостью эффективного управления финансовыми потоками. Традиционные инструменты, такие как электронные таблицы Excel, не обеспечивают достаточного уровня автоматизации, совместной работы и прогнозирования. Существующие корпоративные системы (1С, SAP) избыточно сложны и дороги для небольших компаний.

Веб-системы финансового планирования представляют собой доступную альтернативу, позволяющую автоматизировать процесс составления бюджетов, учёта доходов и расходов, а также построения прогнозов на основе исторических данных. Разработка такой системы требует применения современных архитектурных подходов, обеспечения безопасности данных и реализации точных математических алгоритмов прогнозирования.

Таким образом, актуальность выбранной темы обусловлена необходимостью создания функциональной, масштабируемой и безопасной веб-системы для финансового планирования малого бизнеса.

### Цель и задачи исследования

**Целью курсовой работы** является разработка и реализация веб-системы финансового планирования «ФинГоризонт» на базе PHP и MySQL с учётом требований к функциональности, точности расчётов и безопасности.

**Для достижения поставленной цели** в работе предполагается решение следующих задач:

1. Проанализировать современные системы управления контентом и обосновать выбор технологической платформы для реализации проекта.
2. Рассмотреть архитектурные подходы к созданию финансовых веб-приложений и определить ключевые требования к их функциональной структуре.
3. Спроектировать архитектуру системы «ФинГоризонт» с учётом модульного принципа построения.
4. Реализовать систему аутентификации и авторизации пользователей.
5. Разработать модуль управления сценариями бюджетов (CRUD операции).
6. Создать модуль прогнозирования на основе линейной регрессии с сезонной корректировкой.
7. Провести тестирование разработанного решения и выполнить его оптимизацию.
8. Сформулировать выводы и определить перспективы дальнейшего развития проекта.

### Объект и предмет исследования

**Объектом исследования** являются системы финансового планирования как специализированные информационные системы, ориентированные на автоматизацию бюджетирования и прогнозирования.

## 1. ТЕОРЕТИЧЕСКИЕ ОСНОВЫ РАЗРАБОТКИ СИСТЕМ ФИНАНСОВОГО ПЛАНИРОВАНИЯ

### 1.1 Анализ современных систем управления контентом для создания веб-приложений

#### 1.1.1 Обзор современных систем управления контентом

Система управления контентом (Content Management System, CMS) представляет собой программную платформу, предназначенную для создания, структурирования и публикации цифрового контента в сети Интернет. Однако для разработки специализированных веб-приложений, таких как система финансового планирования, традиционные CMS могут быть избыточны.

Современный рынок предлагает широкий спектр подходов к разработке веб-приложений:

**Фреймворки PHP:**
- Laravel — полнофункциональный фреймворк с поддержкой MVC, ORM, миграций
- Symfony — модульный фреймворк для enterprise-решений
- CodeIgniter — лёгкий фреймворк для быстрых проектов

**Чистый PHP:**
- Прямая разработка без использования фреймворков
- Полный контроль над архитектурой и производительностью
- Минимальные накладные расходы

Для системы «ФинГоризонт» был выбран подход с использованием чистого PHP 8.x, что обусловлено следующими факторами:
- Необходимость полного контроля над алгоритмами расчётов
- Минимизация зависимостей для повышения безопасности
- Оптимальная производительность для вычислительных задач

#### 1.1.2 Сравнительный анализ баз данных

Для хранения финансовых данных критически важен правильный выбор СУБД:

**MySQL/MariaDB:**
- Поддержка транзакций (InnoDB)
- Типы данных DECIMAL для точных финансовых расчётов
- Внешние ключи для обеспечения целостности данных
- Широкое распространение и поддержка

**PostgreSQL:**
- Расширенные возможности для аналитики
- Поддержка оконных функций
- Более сложный в администрировании

**SQLite:**
- Простота развёртывания
- Ограниченная производительность при высоких нагрузках

Для проекта «ФинГоризонт» выбрана MySQL 8.0 благодаря поддержке типа DECIMAL(15,2) для точного хранения денежных сумм и наличию внешних ключей для обеспечения ссылочной целостности.

#### 1.1.3 Обоснование выбора технологического стека

Выбор стека PHP + MySQL обусловлен рядом факторов:

1. **Производительность:** PHP 8.x обеспечивает высокую скорость выполнения скриптов благодаря JIT-компиляции.
2. **Безопасность:** Prepared Statements PDO защищают от SQL-инъекций.
3. **Масштабируемость:** Возможность горизонтального масштабирования через репликацию БД.
4. **Стоимость:** Открытое ПО не требует лицензионных отчислений.
5. **Поддержка:** Активное сообщество и обширная документация.

### 1.2 Архитектурные подходы к созданию финансовых веб-систем

#### 1.2.1 Основные компоненты системы финансового планирования

Система «ФинГоризонт» как специализированное веб-приложение включает несколько взаимосвязанных компонентов:

**Модуль аутентификации:**
- Регистрация пользователей
- Вход по email/паролю
- Управление сессиями
- Хеширование паролей (password_hash)

**Модуль управления сценариями:**
- Создание бюджетных планов
- Установка периодов действия
- Активные/архивные сценарии

**Модуль управления бюджетом:**
- CRUD операции для статей доходов/расходов
- Категоризация статей
- Повторяющиеся статьи
- Фильтрация и поиск

**Модуль прогнозирования:**
- Линейная регрессия для выявления тренда
- Сезонная корректировка
- Расчёт уровня доверия
- Визуализация прогнозов

**Модуль визуализации:**
- Графики Chart.js
- Динамика доходов/расходов
- Сравнение истории и прогноза

### 1.3 Требования к функциональности системы финансового планирования

#### 1.3.1 Функциональные требования

**Система должна обеспечивать:**

1. **Аутентификацию и авторизацию:**
   - Регистрация новых пользователей
   - Вход по email/паролю
   - Сессионная авторизация
   - Защита от CSRF-атак

2. **Управление сценариями:**
   - Создание бюджетных сценариев
   - Редактирование параметров
   - Деактивация/архивирование
   - Удаление сценариев

3. **Управление бюджетом:**
   - Добавление статей доходов и расходов
   - Категоризация статей
   - Повторяющиеся статьи
   - Фильтрация по типу и категории
   - Сводная статистика

4. **Прогнозирование:**
   - Генерация прогноза на 3/6/12 месяцев
   - Алгоритм: линейная регрессия с сезонной корректировкой
   - Расчёт уровня доверия (confidence level)
   - Визуализация графиков

#### 1.3.2 Нефункциональные требования

**Качественные параметры системы:**

1. **Производительность:**
   - Время отклика API < 200 мс
   - Загрузка страниц < 2 секунд
   - Поддержка до 100 одновременных пользователей

2. **Безопасность:**
   - Хеширование паролей (PASSWORD_DEFAULT)
   - Prepared Statements для всех запросов
   - CSRF токены
   - Проверка прав доступа к данным

3. **Точность вычислений:**
   - Использование DECIMAL(15,2) в БД
   - Округление до 2 знаков после запятой
   - Отсутствие потерь точности при расчётах

4. **Юзабилити:**
   - Адаптивный дизайн
   - Интуитивная навигация
   - Валидация форм на клиенте и сервере

5. **Масштабируемость:**
   - Возможность добавления новых модулей
   - Поддержка мультиязычности
   - Экспорт данных в различные форматы

#### 1.3.3 Выбор технических решений для реализации требований

Реализация обозначенных требований осуществляется посредством:

- **PDO (PHP Data Objects)** — для безопасной работы с БД
- **password_hash()/password_verify()** — для хеширования паролей
- **session_start()** — для управления сессиями
- **bin2hex(random_bytes(32))** — для генерации CSRF токенов
- **Chart.js** — для визуализации данных
- **DECIMAL(15,2)** — для хранения денежных сумм

### 2.1 Проектирование архитектуры и выбор технических решений

#### 2.1.1 Архитектурная схема системы «ФинГоризонт»

В соответствии с теоретическими положениями, рассмотренными в первой главе, была разработана архитектурная модель системы «ФинГоризонт». Архитектура построена по трёхуровневому принципу:

**Уровень представления (Frontend):**
- HTML5 шаблоны страниц
- CSS3 стили (style.css)
- Vanilla JavaScript для интерактивности
- Chart.js для графиков

**Уровень приложения (Backend API):**
- api/auth.php — аутентификация

#### 2.3.1 Алгоритм прогнозирования

Файл api/predictions.php реализует алгоритм на основе линейной регрессии с сезонной корректировкой:

**Шаг 1: Получение исторических данных**

**Шаг 2: Расчёт линейного тренда (метод наименьших квадратов)**
```php
function calculateLinearTrend($values) {
    $n = count($values);
    $sumX = array_sum(range(0, $n - 1));
    $sumY = array_sum($values);
    $sumXY = 0;
    $sumX2 = 0;
    
    foreach ($values as $i => $value) {
        $sumXY += $i * $value;
        $sumX2 += $i * $i;
    }
    
    $denominator = ($n * $sumX2 - $sumX * $sumX);
    $slope = $denominator != 0 ? ($n * $sumXY - $sumX * $sumY) / $denominator : 0;
    $intercept = ($sumY - $slope * $sumX) / $n;
    
    return ['slope' => $slope, 'intercept' => $intercept];
}
```

**Шаг 3: Построение сезонных коэффициентов**
```php
function buildSeasonalityMap($historicalData, $field, $average) {
    $result = array_fill(1, 12, 1.0);
    if ($average <= 0) return $result;
    
    $monthBuckets = [];
    foreach ($historicalData as $item) {
        $monthNum = (int)date('n', strtotime($item['month'] . '-01'));
        $monthBuckets[$monthNum][] = (float)$item[$field];
    }
    
    foreach ($monthBuckets as $monthNum => $values) {
        $monthAvg = array_sum($values) / count($values);
        $factor = $monthAvg / $average;
        // Ограничиваем фактор
        $result[$monthNum] = max(0.5, min(1.5, $factor));
    }
    return $result;
}
```

**Шаг 4: Расчёт уровня доверия**
```php
$baseConfidence = min(95, 50 + ($n * 5));
$volatilityPenalty = (($incomeVariation + $expenseVariation) / 2) * 30;
$modelQualityBonus = (($incomeR2 + $expenseR2) / 2) * 10;
$horizonPenalty = $i * 1.5;
$confidenceLevel = max(30, min(95, 
    $baseConfidence - $volatilityPenalty + $modelQualityBonus - $horizonPenalty
));
```

**Шаг 5: Генерация прогноза**
```php
for ($i = 1; $i <= $months; $i++) {
    $forecastDate = clone $lastDate;
    $forecastDate->modify("+{$i} month");
    $forecastMonth = $forecastDate->format('Y-m');
    $monthNum = (int)$forecastDate->format('n');
    
    // Тренд по регрессии
    $incomeByTrend = $incomeTrend['intercept'] + $incomeTrend['slope'] * ($n + $i - 1);
    $expenseByTrend = $expenseTrend['intercept'] + $expenseTrend['slope'] * ($n + $i - 1);
    
    // Сезонная корректировка
    $incomeSeasonalFactor = $incomeSeasonality[$monthNum] ?? 1.0;
    $expenseSeasonalFactor = $expenseSeasonality[$monthNum] ?? 1.0;
    
    $predictedIncome = max(0, $incomeByTrend * $incomeSeasonalFactor);
    $predictedExpense = max(0, $expenseByTrend * $expenseSeasonalFactor);
    $predictedBalance = $predictedIncome - $predictedExpense;
    
    $predictions[] = [
        'month' => $forecastMonth,
        'predicted_income' => round($predictedIncome, 2),
        'predicted_expense' => round($predictedExpense, 2),

}
```

### Приложение 1. Техническое задание

**Разработка веб-системы финансового планирования «ФинГоризонт»**

**1. Общие сведения**

Объект разработки: Веб-приложение для финансового планирования малого бизнеса.

Назначение: Предоставление пользователям возможности создания бюджетных сценариев, учёта доходов/расходов и прогнозирования финансовых показателей.

Платформа: PHP 8.x + MySQL 8.0

**2. Требования к функциональности**

Система должна быть разделена на четыре ключевых функциональных блока:

**2.1. Модуль аутентификации**
- Регистрация по email с паролем (мин. 6 символов)
- Вход с проверкой учётных данных
- Сессионная авторизация
- Хеширование паролей (password_hash)
- CSRF защита

**2.2. Модуль управления сценариями**
- CRUD операции для сценариев
- Поля: название, описание, дата начала, дата окончания, статус активности
- Привязка к пользователю
- Каскадное удаление связанных статей

**2.3. Модуль управления бюджетом**
- CRUD операций для статей доходов/расходов
- Поля: название, тип (income/expense), категория, сумма, дата
- Повторяющиеся статьи (daily/weekly/monthly/yearly)
- Фильтрация по типу и категории
- Сводная статистика

**2.4. Модуль прогнозирования**
- Алгоритм: линейная регрессия + сезонная корректировка
- Горизонт прогноза: 3/6/12 месяцев
- Расчёт confidence level на основе волатильности
- Сохранение прогнозов в БД
- Визуализация через Chart.js

**3. Технические требования**

**База данных:**
- MySQL 8.0 с движком InnoDB
- Тип DECIMAL(15,2) для денежных сумм
- Внешние ключи с ON DELETE CASCADE
- UTF-8 кодировка

**Безопасность:**
- Prepared Statements (PDO)
- password_hash(PASSWORD_DEFAULT)
- CSRF токены
- Проверка прав доступа

**Производительность:**
- Индексы на часто используемых полях
- Оптимизация SQL запросов
- Минимизация HTTP запросов

**4. Состав таблиц базы данных**

**users:**
- id (PK)
- email (UNIQUE)
- password_hash
- company_name
- created_at

**scenarios:**

- end_date
- is_active

**budget_items:**
- id (PK)

- date
- is_recurring
- recurrence_pattern
- notes
- created_at

**predictions:**
- id (PK)
- scenario_id (FK → scenarios)
- prediction_date
- predicted_income (DECIMAL)
- predicted_expense (DECIMAL)
- predicted_balance (DECIMAL)
- confidence_level (DECIMAL)
- algorithm_used
- created_at

**5. Контроль качества**

- Валидация всех входных данных

**Листинг 1: Функция подключения к БД (config.php)**
```php
function getDBConnection() {
    static $pdo = null;
    if ($pdo === null) {
        try {
            $dsn = "mysql:host=" . DB_HOST . ";dbname=" . DB_NAME . ";charset=" . DB_CHARSET;
            $options = [
                PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION,
                PDO::ATTR_DEFAULT_FETCH_MODE => PDO::FETCH_ASSOC,
                PDO::ATTR_EMULATE_PREPARES => false,
            ];
            $pdo = new PDO($dsn, DB_USER, DB_PASS, $options);
        } catch (PDOException $e) {
            error_log("Ошибка подключения к БД: " . $e->getMessage());
            die(json_encode(['success' => false, 'error' => 'Ошибка подключения']));
        }
    }
    return $pdo;
}
```

**Листинг 2: Алгоритм линейной регрессии (predictions.php)**
```php
function calculateLinearTrend($values) {
    $n = count($values);
    if ($n === 0) return ['slope' => 0, 'intercept' => 0];
    
    $sumX = array_sum(range(0, $n - 1));
    $sumY = array_sum($values);
    $sumXY = 0;
    $sumX2 = 0;
    
    foreach ($values as $i => $value) {
        $sumXY += $i * $value;
        $sumX2 += $i * $i;
    }
    
    $denominator = ($n * $sumX2 - $sumX * $sumX);
    $slope = $denominator != 0 ? ($n * $sumXY - $sumX * $sumY) / $denominator : 0;
    $intercept = ($sumY - $slope * $sumX) / $n;
    
    return ['slope' => $slope, 'intercept' => $intercept];
}
```

```php
case 'create':
    $data = json_decode(file_get_contents('php://input'), true);
    
    // Валидация обязательных полей
    $required = ['scenario_id', 'name', 'type', 'amount', 'date'];

    $stmt = $pdo->prepare("SELECT id FROM scenarios WHERE id = ? AND user_id = ?");
    $stmt->execute([$data['scenario_id'], $_SESSION['user_id']]);
    if (!$stmt->fetch()) {
        jsonResponse(['success' => false, 'error' => 'Сценарий не найден']);
    }
    
    // Вставка записи
    $stmt = $pdo->prepare("
        INSERT INTO budget_items
        (scenario_id, name, type, category, amount, date, is_recurring, recurrence_pattern, notes)
        VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?)
```
